# 🏁 [이더봇 마스코트 시스템] — 팀 회의 결과 (사이클 #0)

## 팀원별 의견

### 지민 (PD)
- **조건부 Go.** Phase 1(히어로 walk + Footer idle)은 스코프 통제 가능. 에셋 확보 전 착수 불가.
- 3단계 롤아웃 제안: Phase 1(MVP 3일) → Phase 2(404, 3일) → Phase 3(About 풀 인터랙션, 4일)
- 리스크: 스코프 크립, 에셋 병목, 성능, 접근성, 유지보수
- 질문→미래: HeroBanner 리팩토링 필요 여부?

### 현우 (기획자①·사용자 리서치)
- 개발자 대상 향수+호감 타깃 적합. "발견의 즐거움" 수준으로 설계 필수.
- 페이지별 사용자 스토리 4종 정의. 점진적 공개(3단계) 전략 제안.
- KPI: 마스코트 클릭률 15%, About 체류시간 +30%, 404→메인 복귀 60%
- 질문→미래: CSS Sprite vs Canvas vs Lottie 기술 선택?

### 소연 (기획자②·비즈니스 전략)
- 브랜드 차별화에 효율적이나 핵심 가치제안(AI 자동 글 생성)과 직접 연결 안 됨.
- "CSS+SVG로 5KB 마스코트 만들기" 기술 포스트로 콘텐츠화 제안.
- 톤 조정 필요: "귀여운 포켓몬"보다 "미니멀 로봇" 방향.
- 질문→승호: About 풀 인터랙션이 "뭐지 이건?" 반응 리스크 평가?

### 태준 (백엔드)
- 순수 클라이언트 구현이라 백엔드 부담 제로. SSR/Hydration 불일치가 최대 리스크.
- `dynamic import + ssr: false` 필수, `not-found.tsx` 신규 생성 필요.
- 확장 경계선: Phase 1=휘발성, Phase 2=localStorage, Phase 3=API
- 질문→미래: 파티클을 Canvas vs DOM? SSR 분리 전략 영향.

### 미래 (프론트엔드)
- ~5KB 번들 영향 없어 좋은 전략. Server/Client 경계 재설계가 핵심 과제.
- 컴포넌트 구조: MascotSVG(공유) + 페이지별 래퍼 + useMascotInteraction 훅
- 예상 공수: ~6.75일 (전체)
- About playground가 본래 목적 희석 우려 → 하단 이동 또는 /playground 분리 검토
- 질문→기획/PM: playground 위치 재배치 가능?

### 윤서 (QA)
- 인터랙션 조합이 많아 상태 충돌·타이밍 버그 확률 높음.
- 테스트 케이스 37개+ 도출 (히어로 7, About 8, 404 3, Footer 2, 코나미 4, 접근성 5, 크로스브라우저 6)
- 핵심 리스크: 상태 충돌, 타이머 누수, 모바일 이벤트 충돌
- 질문→미래: 공유 상태 관리 vs 페이지별 독립 구현?

### 다은 (UI)
- 도트 로봇이 AI 블로그 정체성 강화하나 카카오테크 미니멀 원칙과 충돌 가능.
- `#8B5CF6` 보라색이 디자인 시스템 미등록 — 컬러 토큰 추가 필요.
- SVG 색상을 CSS 변수로 제어하는 구조 필수. 반응형·다크모드 대응 미비.
- 질문→미래: useReducer vs XState?

### 승호 (UX)
- 점진적 공개 3-Tier 모델 제안: 패시브 → 단순 인터랙션 → 숨겨진 인터랙션
- 모바일 터치 타겟 44x44px(WCAG 2.5.5) 필수. 롱프레스/드래그 모바일 대응 필요.
- 404/빈 상태/로딩 시 봇 활용 UX 패턴 제안.
- 질문→미래: Canvas vs DOM에서 접근성(aria-live) 대응 비교?

## 주요 합의점

1. **전면 동시 적용 반대** — 8명 전원. Phase 분리(3단계 롤아웃) 필수.
2. **순수 CSS+SVG 경량 구현 지지** — 외부 라이브러리 도입 불필요.
3. **`prefers-reduced-motion` 대응 필수** — 모든 팀원 언급.
4. **About 풀 인터랙션은 About 전용 격리** — 다른 페이지는 클릭+말풍선 정도만.
5. **`not-found.tsx` 신규 생성 필요** — 현재 존재하지 않음.
6. **MascotSVG 공유 컴포넌트 + 페이지별 래퍼** 구조 합의.
7. **"use client" + dynamic import(ssr: false)** 로 Server Component 격리.
